Game Save 02

Game Save

Web app where users can search, discover, and save information about video game titles pulled using the IGDB API (videogame database).

This project involved many firsts such as using React, Redux, making API requests to fetch data from an API and displaying the information, handling tokens, implementing (OAuth) authentication, some basic CRUD operations, adding user accounts with the data stored in firebase as my database and backend.

Search Games

Searching game titles via search or drop down select genre makes an API request to the IGDB database and returns a list of games in a carousel.

View Game info

View game rating, summary, storyline, social links, videos of trailers/gameplay, screenshots, and artwork.

Authentication (OAuth) // Save game Titles

Sign in with user authentication via Google (firebase) and save or remove game titles to or from your list. Game titles get saved to your list. Quick save from carousel or the game information page.

Challenges

Other than trying to keep up with learning React faster than it was updating, I had a number of challenges. However my biggest challenge was handling asynchronous data.

Many of the errors/bugs were simply due to code being written without consideration to async data such as code being executed before data in a useState hook had a chance to update.

The other big challenge, and another first, was working with a more complex API such as Twitch/IGDB's API. Reading and understanding the documentation, testing (using Postman), and implementing the app access token into my code properly took some time and effort. I ran into a CORS error which required a workaround for the IGDB API by setting up a proxy using CORS anywhere with Heroku.

Another big takeaway was the ability to debug code. I feel this project really tested my debugging skills. First and foremost my mistakes taught me to use best practices in the first place to avoid bugs entirely if you can, but when they do happen, work through them logically, use console logs, chrome dev tools/debugger/breakpoints, and more than likely it's a very easy fix with a solution just around the corner.

Overall this project provided me with a wealth of learning experience with hard lessons learnt that I wouldn't forget and was able to carry on going forward.
